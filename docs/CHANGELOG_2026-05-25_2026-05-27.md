# ECHO Next 26.5.27 更新日志

时间范围：2026-05-25 至 2026-05-27  
取材范围：最近提交记录与当前 26.5.27 工作区候选改动。当前仍有未提交内容，涉及 Connect、DLNA、播放状态、低负载模式、资料库扫描与界面整理，最终发布前仍以实际验收结果为准。

## 重点提醒：数播投送功能施工中

数播投送已经进入集中施工阶段，但还不建议把它当成完全稳定的正式能力使用。

这一轮主要在打通 DLNA / UPnP 数播链路：发现局域网设备、生成可播放的本地 HTTP 音频地址、发送封面与元数据、支持基础播放控制、音量、seek、状态轮询和请求调试信息。Connect 页面也重做了更像“数播控制台”的入口，能区分局域网数播、TV、HQPlayer 和 AirPlay。

当前边界也要说清楚：不同品牌数播对格式、封面、Range 请求、seek 和状态回报的兼容差异很大；AirPlay 仍保持门控；HQPlayer 继续作为专业输出方向推进。这个功能会继续按“低风险、可诊断、不影响本机播放”的原则施工。

## 播放体验

- 迷你播放器加入音量控制，并修复主窗口与迷你播放器之间播放队列状态不同步的问题。
- 托盘菜单补上播放控制，窗口隐藏时也能更方便地暂停、继续或切歌。
- 播放栏与播放状态开始识别 Connect / HQPlayer 等外部输出状态，避免外部投送时 UI 仍按本机播放状态误判。
- 首页 Now Playing 信息继续扩展，当前播放曲目的专辑、艺术家和基础元数据展示更完整。
- 修复 gapless 场景里坏文件误触发或误判的问题，减少“文件异常”把连续播放链路带偏的概率。

## 音频稳定性

- ASIO4ALL 兼容处理进一步收敛，目标是只隔离第三方包装层风险，不影响原厂 ASIO 设备路径。
- README 顶部补充排障提示：设备或播放异常时优先开启低负载播放模式，并通过控制台/日志提交有效信息。
- 低负载播放模式继续增强：播放期间远程源后台任务、诊断轮询和部分高频工作会更保守，优先保护播放热路径。
- 音频诊断窗口在增强低负载状态下会降低轮询频率，避免诊断工具本身给播放增加额外压力。

## Connect / 数播 / HQPlayer

- DLNA 发现逻辑增强：按可用 IPv4 网卡发起搜索，减少多网卡环境下漏发现设备的概率。
- DLNA 设备信息更完整：保留 model、manufacturer、presentation URL、UDN 等发现信息，方便判断设备真实身份。
- 最近发现但暂时漏报的 DLNA 设备会短时间保留为“不可用”，避免刷新后列表突然消失。
- 本地 Connect HTTP 服务开始记录最近的音频、封面、转码请求，方便定位数播是否真的拉到了资源。
- 封面投送更兼容：支持把本地或远程封面转成 JPEG，并在 DIDL-Lite 元数据里按 DLNA 缩略图方式声明。
- DLNA 状态轮询加入 TransportInfo / PositionInfo，投送后的播放、暂停、停止、进度更容易同步回 ECHO。
- HQPlayer Connect 路径继续补齐连接测试、状态同步、seek、stop 和失败可见性，避免专业输出失败时“静默卡住”。

## 资料库与扫描

- CUE 扫描继续增强：sidecar `.cue` 可以展开为虚拟曲目，并抑制被 CUE 管理的原始音频重复进入普通曲目列表。
- 嵌入式 CUE 与 sidecar CUE 的处理路径合并得更清晰，后续更容易继续做真实专辑轨道验证。
- 远程源后台任务在播放期间更克制，增强低负载模式下可暂停 metadata / duration backfill 等非必要任务。
- 资料库安全测试继续补强，重点保护扫描、删除、恢复和后台任务边界。

## 界面与交互

- Connect 页面重构为更直观的投送工作台：当前投送、设备列表、设备能力、连接状态和 DLNA Out 调试信息集中展示。
- 局域网设备卡片增加设备类型视觉区分，TV、数播、HQPlayer、AirPlay 不再全部挤成同一类入口。
- 播放设置抽屉补充增强低负载开关，让“保护播放优先”的策略可以显式控制。
- Now Playing、桌面歌词、播放栏和相关样式继续打磨，重点是让外部输出、歌词和播放状态不要互相误导。

## 打包与发布

- 版本号推进到 `26.5.27`。
- Windows 打包链继续围绕内置 FFmpeg 工具链做校验，避免依赖用户机器上的 PATH 环境。
- 最近一次 Windows 打包恢复工作已确认：缺失 bundled `ffmpeg.exe` 时，应按 manifest 固定版本恢复并重新跑 `verify:ffmpeg`。

## 已知边界

- 数播投送仍处于施工中，不同设备的兼容性需要继续用真实硬件验证。
- AirPlay 音频投送仍未开放为静默默认能力，必须等元数据、封面、时长和控制链路验收后再放开。
- Connect / DLNA 当前更适合测试和排障，不建议作为“已经完全稳定”的核心发布卖点。
- 当前工作区仍有未提交改动，这篇日志是候选更新说明，不等同于最终 release note。
